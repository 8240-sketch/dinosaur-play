## 1. Visual Identity Statement

> **一条视觉规则**：「温暖的手工绘本感——像翻开一本孩子最喜欢的恐龙故事书。」

### 支撑原则

| 原则 | 一句话设计测试 | 对应 Pillar |
|------|--------------|------------|
| **温暖优先** | 当两个视觉方案并列时，选择更温暖、更柔和的那个——没有例外 | P2（失败是好玩的路）：温暖 = 无威胁 = 犯错不可怕 |
| **触手可及** | 每个交互元素必须让孩子觉得「我能碰到它」——大、圆、有弹性感 | P1（看不见的学习）：如果按钮看起来像考试题，学习就变得可见了 |
| **表情即叙事** | 角色的情绪 90% 靠表情和肢体传达，不靠文字标签 | P3（声音是成长的日记）：视觉叙事减少阅读依赖，让 4 岁孩子也能理解剧情 |

### 视觉排斥清单（Anti-Visual）

以下视觉风格明确禁止出现在本项目中：

- ❌ 写实 3D 渲染（给孩子压迫感）
- ❌ 像素风（辨识度低，幼儿视觉发育不完善）
- ❌ 霓虹/赛博朋克色调（与温暖基调冲突）
- ❌ 极简主义/冷淡风（缺乏情感温度）
- ❌ 过度装饰/信息密度过高（认知超载）

### 语义色彩规则

| 含义 | 使用颜色 | 非颜色备份 | 无障碍说明 |
|------|---------|-----------|-----------|
| 可交互 | `--accent-dino` (暖橙) | 所有可交互元素同时有图标+形状变化 | 色盲安全：橙色在所有色觉缺陷类型中可辨识 |
| 成功 | `--feedback-success` (暖绿) | 金星图标 + 动画弹跳 | 形状备份：星形图标 |
| 温和反馈 | `--feedback-gentle` (琥珀) | T-Rex confused 表情 + 肢体动画 | 表情备份：非颜色依赖 |
| 文字高亮 | `--highlight-pulse` (亮黄) | TTS 播放时词汇背景发光 | 视觉备份：文字加粗 + 放大 |
| 背景 | `--surface-bg` (暖米白) | — | — |

### 情感色彩映射

| 游戏状态 | 主导色 | 情绪目标 | 能量水平 |
|----------|--------|---------|---------|
| 探索/对话 | `--surface-bg` 暖米白 + `--accent-dino` 暖橙点缀 | 好奇、安全、温暖 | 舒缓 |
| 词汇选择 | `--accent-dino` 暖橙按钮 + `--surface-bg` 背景 | 期待、兴奋 | 中等 |
| 成功/金星 | `--feedback-success` 暖绿 + `--highlight-pulse` 亮黄 | 骄傲、喜悦 | 高涨 |
| confused/重试 | `--feedback-gentle` 琥珀 + 角色滑稽动画 | 好笑、轻松 | 轻快 |
| 孵化仪式 | `--highlight-pulse` 亮黄光晕 + `--postcard-bg` 暖色 | 神奇、惊喜 | 逐渐升高 |
| 家长地图 | `--surface-bg` 暖米白 + `--feedback-success` 暖绿金星 | 平静、满足 | 舒缓 |

### 色盲安全审计

| 色觉缺陷类型 | 影响的语义色 | 缓解方案 |
|------------|-----------|---------|
| 红色盲 (Protanopia) | 成功绿 vs 背景 | 星形图标 + 弹跳动画提供非颜色线索 |
| 绿色盲 (Deuteranopia) | 成功绿 vs 琥珀反馈 | confused 动画的表情差异远大于颜色差异 |
| 蓝色盲 (Tritanopia) | 无显著影响 | 橙/黄/绿在此色觉缺陷下仍可区分 |

## 3. Lighting & Atmosphere

### 整体光照方向

- **光源方向**：左上方 45° 柔和光（模拟自然日光从窗户照入）
- **色温**：暖色调为主（3200K–4500K 等效），无冷蓝光
- **对比度**：低至中等——阴影柔和，无硬边阴影
- **氛围关键词**：温暖、安全、绘本、柔和、亲切

### 各场景氛围定义

| 场景 | 情绪目标 | 光照特征 | 氛围形容词 | 关键视觉元素 |
|------|---------|---------|-----------|------------|
| **Main Menu** | 归属感、期待 | 温暖漫射光，T-Rex 身上有柔和高光 | 舒适、等待、友好 | T-Rex 闲置动画 + 暖色背景 |
| **HatchScene** | 神奇、惊喜 | 从蛋壳缝隙透出的亮黄光芒，逐渐增强 | 神秘、温暖、期待 | 蛋壳裂缝光效 + CPUParticles2D 暖色粒子 |
| **ChoiceUI** | 专注、兴奋 | 均匀柔光，选项按钮有微妙高光 | 清晰、鼓励、温暖 | 两个暖橙按钮 + 图标发光 |
| **Dialogue/Narrative** | 沉浸、好奇 | 场景环境光，NPC 身上有焦点光 | 故事感、亲密 | T-Rex 表情 + 文字气泡 |
| **VocabMap (Parent)** | 平静、满足 | 柔和均匀光，无强烈高光 | 收获、回忆 | 金星图标 + 暖色词汇卡 |
| **Postcard** | 分享、骄傲 | 明亮温暖光，轻微过曝感 | 纪念、温暖 | 恐龙场景定格 + 暖色边框 |

### 禁止的光照效果

- ❌ 冷色光/蓝色调（制造紧张感）
- ❌ 硬边阴影（给孩子压迫感）
- ❌ 闪烁/频闪光效（光敏性癫痫风险）
- ❌ 过度泛光/Bloom（分散注意力）

## 4. Shape Language

### 角色剪影哲学

- **圆润为王**：所有角色轮廓以圆形和椭圆形为主，避免尖锐角
- **头身比**：2:1 到 2.5:1（大头 = 可爱 = 可亲近）——T-Rex 头部占身体 40%
- **可辨识度**：每个恐龙角色在纯黑剪影状态下必须可区分
  - T-Rex：大头 + 小短手 + 粗腿
  - Triceratops：三角头冠 + 四肢均匀
  - 蛋：椭圆 + 斑点纹理

### 环境几何语言

| 元素类型 | 形状语言 | 原因 |
|----------|---------|------|
| 地面/地形 | 平缓曲线、圆润土丘 | 安全感——没有尖锐的悬崖或锯齿边缘 |
| 植被 | 圆形树冠、粗壮树干 | 绘本感——像孩子画的树 |
| 建筑/结构 | 无（恐龙世界无建筑） | 自然环境为主 |
| 岩石 | 圆角多边形、光滑表面 | 避免尖锐感 |

### UI 形状语法

- **按钮**：圆角矩形（圆角半径 ≥ 16dp）——像柔软的枕头
- **对话框**：圆角矩形 + 柔和阴影——像绘本中的文字气泡
- **卡片**（词汇卡、明信片）：圆角矩形 + 1px 细边框
- **弹窗**：居中 + 圆角 + 半透明背景遮罩
- **所有 UI 元素禁止直角**——直角 = 冷 = 不友好

### 英雄形状 vs 支撑形状

| 类别 | 形状特征 | 视觉权重 |
|------|---------|---------|
| **英雄形状**（角色、选项按钮） | 大、圆、有表情、暖色填充 | 高——视觉焦点 |
| **支撑形状**（背景、装饰、边框） | 小、柔和、低饱和度 | 低——退居二线 |

## 5. Character Art Direction

### 视觉原型

| 角色 | 视觉原型 | 关键特征 | 比例 |
|------|---------|---------|------|
| **T-Rex** | 大头萌宠恐龙 | 大头（占体 40%）、圆眼睛、小短手、粗腿、暖绿色皮肤 | 头身比 2.5:1 |
| **Triceratops** | 稳重小伙伴 | 三角头冠、圆润身体、温和表情、蓝灰色皮肤 | 头身比 2:1 |
| **蛋（孵化前）** | 神秘宝物 | 椭圆、暖色斑点、微微发光、有生命感 | — |

### 表情/姿态风格

- **表情风格**：夸张但不恐怖——眼睛占头部 30%，嘴巴大但圆润
- **姿态风格**：弹性动画（挤压拉伸原则），动作幅度大但节奏舒缓
- **confused 表情**：眼睛变成圈圈、嘴巴歪向一边、身体微微摇晃——好笑而非沮丧
- **happy 表情**：眼睛弯成月牙、嘴巴大张、身体弹跳——孩子会跟着笑
- **角色永远不出现**：恐惧、愤怒、哭泣、受伤的表情

### LOD 哲学

| 距离 | 细节级别 | 说明 |
|------|---------|------|
| 近景（对话/选择） | 全细节：眼睛高光、皮肤纹理、表情微动 | 角色占屏幕 40-60% |
| 中景（场景过渡） | 简化纹理，保留表情轮廓 | 角色占屏幕 20-30% |
| 远景（背景装饰） | 纯色剪影 + 轮廓 | 角色占屏幕 <10% |

## 6. Environment Design Language

### 整体风格

- **风格定位**：手绘绘本风——像 Eric Carle（《好饿的毛毛虫》作者）的拼贴质感
- **纹理哲学**：有纹理但不写实——纸张纹理叠加在色块上，制造手工感
- **色彩饱和度**：中等偏高——孩子喜欢鲜艳色但不能刺眼

### 场景设计规则

| 场景 | 环境特征 | 关键元素 | 情绪 |
|------|---------|---------|------|
| **恐龙森林** | 圆形树冠、粗壮树干、柔和草地、远处圆山 | 巨大蘑菇、圆石头、柔和阳光 | 安全、好奇 |
| **孵化洞穴** | 温暖岩石、柔光、蛋巢 | 发光的蛋、柔和阴影、温暖色调 | 神秘、期待 |
| **草地/空地** | 平缓起伏的绿色草地、蓝天白云 | 野花、圆石头、远处恐龙轮廓 | 开阔、自由 |

### 道具密度规则

- **场景密度**：稀疏——每个场景 3-5 个可识别元素，不超过 8 个
- **原则**：幼儿视觉追踪能力有限，场景信息密度必须低于成人游戏 50%
- **焦点引导**：NPC 和选项按钮是唯一高饱和度元素，环境元素饱和度降低 30%

## 7. UI Visual Language

### UI 风格方向

- **风格**：非写实、绘本风延伸——UI 是故事书的一部分，不是独立的 HUD
- **字体**：Nunito（圆润、友好）+ Noto（CJK 补全）
- **字号层级**：
  - 英文单词：28sp 加粗（最高视觉权重）
  - 中文翻译：14sp 常规（灰色，低权重）
  - 对话文字：20sp 常规
  - 按钮标签：22sp 加粗

### 图标风格

- **风格**：线条图标 + 暖色填充——圆润、简洁、无写实细节
- **线条粗细**：2-3dp，圆角端点
- **填充**：半透明暖色填充 + 实色描边
- **禁止**：写实图标、线性图标（太冷）、像素图标

### UI 动画感觉

- **按钮反馈**：按下时弹性缩小（0.95x）→ 释放时回弹（1.05x → 1.0x）——像按下一个软枕头
- **出现动画**：从下方轻微弹入（200ms ease-out）——不是生硬出现
- **消失动画**：淡出 + 轻微缩小（150ms）——优雅退场
- **禁止**：突然出现/消失、剧烈晃动、旋转

### 布局原则

- **竖屏 360×800dp**：所有内容垂直排列，无水平滚动
- **安全区域**：顶部 48dp + 底部 48dp 留空（状态栏 + 导航栏）
- **内容区域**：中间 704dp 高度用于游戏内容
- **触控友好**：所有可交互元素间距 ≥ 16dp，最小触控目标 96dp

## 8. VFX & Particle Style

### 粒子风格

- **风格**：柔和、圆形粒子——像彩色纸屑或光点
- **颜色**：使用调色板中的暖色（橙、黄、绿），禁止冷色粒子
- **运动**：缓慢、有弹性——重力轻、有随机扰动
- **尺寸**：8-24dp，圆形或星形

### 场景特效

| 特效 | 实现方式 | 视觉描述 |
|------|---------|---------|
| **词汇选中** | Tween + 高亮脉冲 | 词汇背景发光 0.5s，亮度从 100% → 150% → 100% |
| **金星获得** | CPUParticles2D | 金色星形粒子从中心向外扩散，缓慢上浮消散 |
| **蛋壳裂缝** | CPUParticles2D + Shader | 暖黄色光从裂缝透出，粒子向外散射 |
| **TTS 播放** | Shader + Tween | 词汇背景亮黄脉冲，跟随语音节奏 |
| **confused 动画** | AnimationPlayer | T-Rex 头部摇晃 + 眼睛变圈圈 + 身体轻微弹跳 |

### 禁止的特效

- ❌ 闪光/频闪（光敏性癫痫风险，3Hz 以上禁止）
- ❌ 粒子爆炸/碎片飞溅（给孩子惊吓感）
- ❌ 屏幕震动（幼儿设备持握不稳）
- ❌ 快速旋转粒子（分散注意力）

## 9. Asset Standards

### 文件命名规范

```
[category]_[name]_[variant]_[size].[ext]

示例：
char_trex_happy_512.png
char_triceratops_idle_512.png
env_forest_tree_01_256.png
ui_btn_primary_96.png
vfx_star_particle_32.png
```

### 纹理规格

| 类别 | 最大分辨率 | 格式 | 色彩空间 | 说明 |
|------|-----------|------|---------|------|
| 角色（近景） | 512×512 | PNG (RGBA) | sRGB | 带透明通道 |
| 角色（中远景） | 256×256 | PNG (RGBA) | sRGB | 简化版 |
| 环境元素 | 256×256 | PNG (RGBA) | sRGB | 树、石头等 |
| UI 元素 | 128×128 | PNG (RGBA) | sRGB | 按钮、图标 |
| 粒子纹理 | 32×32 | PNG (RGBA) | sRGB | 圆形/星形 |
| 明信片模板 | 1080×1080 | PNG | sRGB | 固定尺寸 |
| 背景 | 720×1280 | PNG 或 WebP | sRGB | 竖屏全屏 |

### 内存预算

| 类别 | 单纹理上限 | 场景总纹理上限 |
|------|-----------|--------------|
| 角色 | 512KB | 2MB |
| 环境 | 256KB | 1MB |
| UI | 128KB | 512KB |
| 粒子 | 32KB | 128KB |

### 动画标准

| 参数 | 值 | 说明 |
|------|-----|------|
| NPC 动画帧率 | 12fps | 足够流畅且节省内存 |
| UI 过渡帧率 | 60fps | 跟随引擎帧率 |
| 动画混合时间 | 200-300ms | 平滑过渡，不突兀 |
| Sprite Sheet 布局 | 网格式 | 每行一个动画状态 |

### 导出设置

- **PNG**: 无压缩，保留 Alpha 通道
- **WebP**: 仅用于背景图（有损压缩，质量 85%）
- **色彩配置文件**: sRGB（Android 标准）
- **DPI**: 160dpi（mdpi 基准，Godot 自动缩放）

## 10. Style Prohibitions

### 绝对禁止

| 禁止项 | 原因 | 替代方案 |
|--------|------|---------|
| 红色错误反馈 | P2 Anti-Pillar：红色 = 恐惧 | 暖橙/琥珀 + confused 动画 |
| 测验式界面 | P1 Anti-Pillar：学习不可见 | 叙事驱动选择 |
| 写实渲染 | 给孩子压迫感 | 手绘绘本风 |
| 倒计时/时间压力 | P2 Anti-Pillar：无压力 | 无时间限制 |
| 尖锐几何形状 | 不友好、有威胁感 | 圆角、曲线 |
| 冷色调 | 不温暖 | 暖色调为主 |
| 闪烁/频闪 | 光敏性癫痫风险 | 柔和过渡 |
| 信息密度过高 | 幼儿认知超载 | 稀疏布局，每屏 ≤5 元素 |
| 复杂导航 | 幼儿无法理解 | 单层导航，大按钮 |
| 文字标签替代视觉 | 4 岁孩子不识字 | 图标 + 表情 + TTS |

### 需要审批的例外

| 场景 | 可能的例外 | 审批条件 |
|------|-----------|---------|
| 第 2 章预告 | 可能引入新色调 | 必须保持暖色调基底 |
| 家长地图 | 可能信息密度更高 | 面向家长，非幼儿操作 |
