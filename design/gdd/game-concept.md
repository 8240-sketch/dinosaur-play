# Game Concept: 恐龙叙事英语启蒙游戏

> **状态**: 已锁定（4 周时间线，v1 范围）
> **最后更新**: 2026-05-05（从 CEO Plan v2 提炼）
> **CEO 计划**: `.gstack/projects/zhangspaghetti-baby-play/ceo-plans/2026-05-01-english-narrative-game.md`

---

## 1. Overview

面向 4–6 岁中文母语孩子的恐龙主题英语叙事游戏。孩子通过触屏选择英文单词图标来推动剧情：T-Rex 听不懂，孩子来教它——一关 5–10 分钟，通关后生成可分享恐龙明信片。家长看到的不是分数，是孩子第一次说出「T-Rex!」的声音录音和词汇地图里的金星。

**平台**: Android（API 24+）
**受众**: 4–6 岁儿童（主操作者）+ 家长（见证者）
**游戏时长**: 单关 5–10 分钟
**商业模式**: v1 完全免费，APK 直发

---

## 2. Player Fantasy

> **孩子的视角**：「我教恐龙说英文，恐龙听懂了我。」

孩子是世界的钥匙。每一个英文单词是打开下一扇门的钥匙，不是一道考题。选错了没有红叉——T-Rex 做出滑稽的「听不懂」动作，然后挥手离开；这是另一条好玩的路，不是失败。

> **家长的视角**：「我们家孩子已经学会了 T-Rex 等 5 个英文单词。」

明信片发朋友圈，词汇地图里有金星，录音回放能听到孩子六个月前的声音——学习过程被见证，家长骄傲有据可依，无需理解数字。

---

## 3. Core Pillars

| 编号 | Pillar | 一句话说明 |
|------|--------|----------|
| P1 | **看不见的学习** | 界面里没有「学英语」，只有「你看到了什么？」——孩子为推进剧情而选单词，不为答题 |
| P2 | **失败是另一条好玩的路** | confused 动画要好笑，重试按钮要鼓励，无红色、无哭脸、无倒计时压力 |
| P3 | **声音是成长的日记** | 孩子的第一声「T-Rex!」被 App 悄悄录下；六个月后家长回放，听到的是幼稚清亮的声音 |
| P4 | **家长是骄傲见证者，不是监工** | 词汇金星不需要解释，明信片直接分享，家长模式是可选的，不是必须的 |

### Anti-Pillars（明确不做）

- ❌ 测验式界面（红对勾/红叉，计分板，「你答错了」）
- ❌ 计时压力（倒计时条，时间限制）
- ❌ 红色错误反馈（用 `--feedback-gentle` 暖橙替代）
- ❌ 强制录音（录音是邀请，拒绝权限后功能静默禁用，剧情不阻断）
- ❌ 强制家长登录（家长地图是可选的，长按 5 秒入口，v1 无 PIN）

## 4. Core Loop

```
30 秒触屏选英文图标（视觉二选一）
    ↓
NPC 即时反应：happy_X 动画（3种随机）+ TTS 发音
    ↓
录音邀请（可选）：T-Rex 举爪，孩子可选择「说说看！Say it!」
    ↓
5 分钟通关（5 词汇 × 3 分支点 × 2 结局）
    ↓
Session 结束：词汇金星 + 恐龙明信片（可分享）+ 第2章预告
    ↓
长期：章节解锁 + 词汇地图 + 录音回放
```

**失败子循环**：错误选项 → confused_X 动画（好笑）→ 「再试一次」鼓励按钮 → 重试（同词汇，同场景）

## 5. Unique Hook

**「教恐龙说英文」而不是「学英语 App」**

同类产品（洪恩英语、宝宝巴士）使用显式学习界面（积分、进度条、徽章）。本游戏使用叙事推进作为唯一驱动力——孩子不会说「我在学英语」，只会说「我还要玩」。

关键差异点：
- 选错不扣分，confused 动画是彩蛋
- 没有「学习模式」入口，只有「出发冒险！」
- 通关产出物是明信片（社交媒体可分享），不是「你获得了 100 分」
- 词汇地图面向家长，不出现在孩子视角

## 6. Chapter 1 Narrative Design

**主题**: 恐龙世界

**目标词汇**（5 个）:
| 词汇 | 类别 | 配套 NPC |
|------|------|---------|
| T-Rex | 名词 | T-Rex |
| Triceratops | 名词 | Triceratops |
| eat | 动词 | T-Rex |
| run | 动词 | Triceratops |
| big | 形容词 | T-Rex |

**剧情结构**:
```
开场（孵化/问候）
  → 遇到恐龙（选 T-Rex / Triceratops）
  → 恐龙行为（选 eat / run）
  → 关键分支（big）
  → 结局 A：朋友结局（T-Rex 庆祝 + 明信片）
  → 结局 B：趣味逃跑（T-Rex 远处挥手 + 场景闪回 + 主菜单）
```

**分支数**: 3 个选项分支点
**结局数**: 2 种（A/B 均为正向情绪，无真正「失败」结局）

## 7. First-Launch Flow

```
首次启动
  → HatchScene.tscn（孵化彩蛋：蛋轻颤 → 触屏 → 破壳 → T-Rex 探头）
  → 名字输入页（全屏，可跳过，默认「小朋友」，最大 20 字符）
  → 主菜单（T-Rex 闲置动画，进入游戏大按钮）

常规启动（times_played > 0）
  → 直接主菜单（T-Rex 开场认出孩子：trex_recognize 动画）
```

## 9. Technology Stack

| 层级 | 选择 | 版本 |
|------|------|------|
| 引擎 | Godot Standard（非 Mono） | 4.6 |
| 叙事运行时 | inkgd（GDScript 原生） | github.com/ephread/inkgd |
| 叙事语言 | Ink | 最新 |
| 代码语言 | GDScript | 静态类型 |
| 目标平台 | Android | API 24+（Android 7.0）|
| TTS | DisplayServer.tts_speak() | 系统内置，降级文字高亮 |
| 录音 | AudioStreamMicrophone + AudioEffectCapture | GDScript 路径，无 JNI |
| 存储 | Godot SaveGame JSON（本地） | schema v2（多档案）|

**P0 备选**（inkgd 第1周末验证失败时）: 自建轻量 JSON 叙事状态机（~150 行 GDScript）。

## 10. Visual Identity

### 色彩令牌

| 令牌 | 用途 | 禁忌 |
|------|------|------|
| `--surface-bg` | 主背景，暖米白 | 禁用纯白 |
| `--text-primary` | 主文字，近黑带暖 | — |
| `--text-secondary` | 中文翻译，灰色 | — |
| `--accent-dino` | 暖橙，**仅用于交互元素** | 禁止用作背景色 |
| `--highlight-pulse` | TTS 激活发光，亮黄 | — |
| `--feedback-success` | 答对，暖绿 | — |
| `--feedback-gentle` | 答错，暖橙/琥珀 | **禁用红色** |
| `--postcard-bg` | 明信片背景基调 | — |

### 字体

**Nunito**（Google Fonts，开源）：圆润感与儿童游戏调性匹配，CJK 用 Noto 补全。

### 触控规格

- 选项按钮：最小 **96dp 高 × 屏幕宽度 44–46%**（竖屏双选项并排，中间 8–12% 间距；三区布局图标+英文+中文实测最小高度为 96dp，覆盖 WCAG 最低触摸目标 44dp）
- 按钮内层级：图标（≥60% 面积，顶部）→ 英文单词（20sp 加粗）→ 中文翻译（10sp，灰色）
- 录音按钮：**96dp**+ 大圆形，`--accent-dino` 橙色（更新自 80dp；RecordingInviteUI GDD CD-GDD-ALIGN D-3 2026-05-08）
- 对话文字：最小 18sp，英文单词加粗
- 全程仅点击，无滑动/拖拽/捏合

## 11. Acceptance Criteria（v1 完成定义）

- [ ] Android APK 可在真机安装并运行（API 24+）
- [ ] inkgd 可正常推进 ink 对话、选项和 tag 解析
- [ ] 全程通关两条结局（A/B）无崩溃
- [ ] 词汇地图在通关后正确显示所有词汇数据
- [ ] 恐龙明信片可生成并分享（1080×1080 模板）
- [ ] 4 岁孩子可独立通关（第4周侄女 + 2–3 名孩子用户测试）
- [ ] 失败分支让孩子笑，不哭
- [ ] 通关后孩子主动要求再玩
